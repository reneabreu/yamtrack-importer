## 2. New conventions / vocabulary

*None of this is in CLAUDE.md yet — CLAUDE.md currently holds only the lz-playbook block, no project/architecture context. Recommend adding a slim "Architecture" section to CLAUDE.md (sources→resolve→exporter seam; library; storage roots) so these stop living only in handoffs.*

- **Storage roots** (`webapp/config.py`): `CONFIG_DIR` (settings/overrides), `DATA_DIR` (library.db + history), `MEDIA_DIR` (TMDB/MAL/Crunchyroll caches). Empty env value = unset → defaults to `DATA_DIR`.
- **Library identity + merge** (`core/library.py`, `core/merge.py`): titles keyed by canonical id (anime→mal, tv/movie→tmdb, fallback title); `merge_items` = smart auto-merge (union episodes, max progress/score, status priority, widest dates, unioned `sources`).
- **Delta exports**: `export_snapshots` table + `item_fingerprint`; `export_library(delta=True)` emits only titles changed since that exporter's last export and advances the baseline.
- **Yamtrack is file-only** — no push. The generic `Exporter.push`/api-mode seam is intentionally dormant for a future real-API destination.
- **`.issues-tea/` is gitignored** — it mirrors PRIVATE Gitea issues and must never reach the public GitHub mirror.
